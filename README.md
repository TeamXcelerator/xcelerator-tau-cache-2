# xcelerator-tau-cache-2

Consolidated, public cache of **CCM Weil-form τ-matrices** used across
Team Xcelerator's high-precision spectral-triple work.

> **Author:** Ronnie Andrews, Jr. (Team Xcelerator Inc.®)  
> **License:** See `LICENSE`. Public for academic verification and
> reproduction; redistribution and commercial use restricted.

This is a continuation of
[`xcelerator-tau-cache`](https://github.com/TeamXcelerator/xcelerator-tau-cache),
which reached its storage limit. Both repositories share an identical
layout and file format. Toolkit versions prior to the introduction of
this repo will only check `xcelerator-tau-cache`; newer toolkit versions
check both repositories (this one first, then the original as fallback).

---

## What this is

A CCM τ-matrix is uniquely determined by three integers:

- `lambda_sq` (λ²) — the CCM prime-cutoff parameter
- `nmodes` (N) — the mode cutoff (matrix dimension is `2N+1`)
- `prec` — the working precision, **in bits**

The matrix is **project-agnostic**: for a given `(λ², N, prec)` it is
the same Weil-form construction regardless of which computation
produced it. That universality is why these caches can be consolidated
into a single shared repository.

Each matrix is generated by the Xcelerator Toolkit
(`xc-spectral::ccm::hp`) and stored as a deflated zip of the canonical
JSON form (a flat array of `(2N+1)²` decimal strings).

## Why it exists

τ-matrices are expensive to build (HP Gauss–Legendre integration +
prime-power sums at thousands of digits, minutes to hours per config)
but compress well. Recomputing them on every fresh cloud run wasted
large amounts of compute. This repo lets any run **fetch only the
matrices it needs** — by deterministic URL, no clone required — and
skip the rebuild.

## Layout

Files are organized **precision-first, then by λ², then bucketed by
nmodes** in thousands — consistent with the sibling
[`xcelerator-gl-cache`](https://github.com/TeamXcelerator/xcelerator-gl-cache)
repo, extended for τ's extra λ² dimension. No directory exceeds
GitHub's web-UI listing limit and the path is deterministically
computable from `(λ², N, prec)`:

```
tau_cache/
  prec{P}/
    lambda_sq{L}/
      nmodes{B}-{B+999}/
        lambda_sq{L}_nmodes{N}_prec{P}.json.zip          (single, ≤90 MB)
        lambda_sq{L}_nmodes{N}_prec{P}.json.zip.part00   (byte-split, >90 MB)
        lambda_sq{L}_nmodes{N}_prec{P}.json.zip.part01
        ...
```

where:
- `P` = precision in bits (e.g. `3338` for HP-1000, `6660` for HP-2000)
- `L` = λ² (the prime-cutoff parameter)
- `N` = mode count (matrix dimension `2N+1`)
- `B` = `floor(N / 1000) * 1000` (the lower edge of N's thousand-bucket)

**Example:** the HP-1000 (`prec=3338`) τ-matrix for λ²=1000, N=800
lives at

```
tau_cache/prec3338/lambda_sq1000/nmodes0-999/lambda_sq1000_nmodes800_prec3338.json.zip.part00
tau_cache/prec3338/lambda_sq1000/nmodes0-999/lambda_sq1000_nmodes800_prec3338.json.zip.part01
...
```

### Byte-split parts

Configs whose compressed payload exceeds **90 MB** (under GitHub's
100 MB hard file limit) are byte-split into `.zip.part00`, `.part01`,
… in 90 MB chunks. All parts of a config live in the same folder.
To reconstruct: concatenate the parts in lexicographic order, then
treat the result as one zip:

```bash
cat lambda_sq1000_nmodes800_prec3338.json.zip.part* > combined.json.zip
```

The Xcelerator Toolkit does this automatically on a local cache hit.

### Path formula (for tooling)

```
tau_cache/prec{P}/lambda_sq{L}/nmodes{floor(N/1000)*1000}-{floor(N/1000)*1000+999}/lambda_sq{L}_nmodes{N}_prec{P}.json.zip[.partXX]
```

## Fetching (no clone)

This repo is public, so any file is one raw URL, e.g.:

```bash
curl -sSL -O \
  https://raw.githubusercontent.com/TeamXcelerator/xcelerator-tau-cache-2/main/tau_cache/prec3338/lambda_sq400/nmodes0-999/lambda_sq400_nmodes400_prec3338.json.zip.part00
```

For byte-split configs, fetch each `.partXX` and concatenate. The part
count is not encoded in any single filename; tooling should probe
`.part00`, `.part01`, … until a fetch 404s.

## File format

Each `.json.zip` is a deflate archive containing a single entry named
`lambda_sq{L}_nmodes{N}_prec{P}.json`: a JSON array of `(2N+1)²`
decimal strings (the row-major τ-matrix) at the stated precision. The
Xcelerator Toolkit reads/writes this format directly and structurally
validates it on load (length = `(2N+1)²`, no NaN/Inf, symmetry
`τ[i,j] = τ[j,i]` to working precision); consumers should rely on that
validation rather than assuming any file is correct.
